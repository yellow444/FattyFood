Description models

Description models let you apply common elements (text or HTML) for all category descriptions and images. If you want to display, in addition to or instead of the description of each class or image, the same text or even HTML, this is possible thanks to the description models. The interest of the description models is that you have at your disposal several "variables" containing category and image information as well as conditionals that test the presence of some of this information.

Available variables are : {DESCRIPTION}, {ID}, {PLACE}, {TITLE}, {URL} for categories and images as well as {FILENAME}, {HEIGHT} and {WIDTH} for images only. Conditional structures available are {IF(DESCRIPTION)}...{ENDIF(DESCRIPTION)} et {IF(PLACE)}...{ENDIF(PLACE)}.

Here is an example of a description model for categories :

{IF(DESCRIPTION)}<a href="javascript:;" rel="category_description_{ID}" class="show_parts js_link">Lire la description</a>
<span id="category_description_{ID}" style="display:none">
{DESCRIPTION}</span>{ENDIF(DESCRIPTION)}
